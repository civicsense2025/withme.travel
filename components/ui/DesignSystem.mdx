# withme.travel Design System

Welcome to the withme.travel Design System. This comprehensive resource provides the guidelines, components, and patterns needed to build consistent, accessible, and beautiful user experiences within our travel planning platform.

## Design Principles

Our design system is built around five core principles:

### 1. Intuitive & Accessible

Our interfaces should be easy to understand and use for everyone, including users with disabilities. We design with accessibility in mind from the start, not as an afterthought.

### 2. Collaborative by Design

Our platform is built for collaboration, and our design system reflects this by ensuring components work well in collaborative contexts and support multi-user interactions.

### 3. Responsive & Adaptable

Our components adapt seamlessly across devices and screen sizes, ensuring a consistent experience whether on desktop, tablet, or mobile.

### 4. Travel-Focused

Our design system is tailored specifically for travel planning, with components and patterns that address the unique needs and contexts of travel experiences.

### 5. Consistent & Cohesive

We maintain visual and functional consistency across the platform, using a unified design language that creates a cohesive experience.

## Accessibility Guidelines

Our design system follows WCAG 2.1 AA standards to ensure our platform is accessible to all users.

### Key Requirements

- **Color Contrast**: Maintain a minimum contrast ratio of 4.5:1 for normal text and 3:1 for large text
- **Keyboard Navigation**: All interactive elements must be accessible via keyboard
- **Screen Readers**: Components include appropriate ARIA attributes and semantic HTML
- **Focus Indicators**: Visible focus states for keyboard navigation
- **Text Resizing**: Interface remains usable when text is resized up to 200%
- **Motion & Animation**: Provide options to reduce motion where applicable

### Accessibility Checklist

When creating components, ensure they meet these criteria:

- [ ] Uses semantic HTML elements
- [ ] Includes appropriate ARIA attributes
- [ ] Maintains sufficient color contrast
- [ ] Supports keyboard navigation
- [ ] Works with screen readers
- [ ] Provides visible focus states
- [ ] Handles text resizing appropriately
- [ ] Respects user motion preferences
- [ ] Includes alt text for images and icons

## Theming

Our design system supports both light and dark modes through CSS variables and Tailwind themes.

### Theme Implementation

Components should:

1. Use color tokens from our theme system rather than hardcoded values
2. Test in both light and dark modes
3. Ensure sufficient contrast in both modes
4. Use the `useTheme` hook when needed for theme detection

### Color Usage Guidelines

- Use primary colors for main actions and brand elements
- Use accent colors sparingly for highlighting important information
- Use semantic colors consistently for status and feedback (error, success, etc.)
- Use neutral colors for most UI elements and text

## Getting Started with Development

### Using Components

Import components from their respective locations:

```tsx
import { Button } from '@/components/ui/button';
import { Input } from '@/components/ui/input';
import { Card } from '@/components/ui/card';

function MyComponent() {
  return (
    <Card>
      <h2>Example Form</h2>
      <Input placeholder="Enter your name" />
      <Button>Submit</Button>
    </Card>
  );
}
```

### Creating New Components

When creating new components:

1. Follow the existing directory structure
2. Use TypeScript for type safety
3. Include comprehensive prop documentation
4. Create stories for all variants and states
5. Ensure accessibility compliance
6. Test in both light and dark modes
7. Make components responsive by default
