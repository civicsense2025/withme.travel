# Migration Guide

This guide helps you migrate from our previous component architecture to the new design system.

## Core UI Component Changes

### Button Component

**Before:**
```jsx
<Button type="primary" size="large" onClick={handleClick}>
  Submit
</Button>
```

**After:**
```jsx
<Button variant="primary" size="lg" onClick={handleClick}>
  Submit
</Button>
```

### Input Component

**Before:**
```jsx
<Input type="text" placeholder="Enter name" error={hasError} />
```

**After:**
```jsx
<Input 
  type="text" 
  placeholder="Enter name" 
  variant={hasError ? "error" : "default"} 
/>
```

## Layout Component Changes

### Container Component

**Before:**
```jsx
<Container fluid={true}>
  Content goes here
</Container>
```

**After:**
```jsx
<Container maxWidth="full">
  Content goes here
</Container>
```

### Card Component

**Before:**
```jsx
<Card title="Card Title" bordered>
  <CardContent>Content here</CardContent>
  <CardActions>
    <Button>Action</Button>
  </CardActions>
</Card>
```

**After:**
```jsx
<Card>
  <CardHeader>
    <CardTitle>Card Title</CardTitle>
  </CardHeader>
  <CardContent>Content here</CardContent>
  <CardFooter>
    <Button>Action</Button>
  </CardFooter>
</Card>
```

## Migration Steps

1. **Update dependencies**
   - Ensure you're using the latest version of our component library

2. **Identify components to update**
   - Use our component audit tool to identify all instances of legacy components

3. **Replace one component type at a time**
   - Start with simple components like Button and Text
   - Progress to more complex components like forms and layouts

4. **Test thoroughly**
   - Verify functionality and appearance after each component migration
   - Pay special attention to responsive behavior

5. **Update documentation**
   - Update any internal documentation to reference the new component APIs
