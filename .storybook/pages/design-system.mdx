# withme.travel Design System

Welcome to the withme.travel Design System. This system provides the foundation for creating consistent, accessible, and beautiful user experiences across our travel planning platform.

Our design system is built on a set of core foundations that inform all of our components and patterns.

#### Color Applications

- **Text Colors**: Use grayscale for most text, with primary colors for emphasis or interactive elements
- **Background Colors**: Use light grays for subtle backgrounds, white for content areas

Our typography system is designed for clarity and readability across all devices. We use a primary sans-serif font for all UI elements and text.

#### Font Stack

- **Primary**: "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif

| ----- | ------- | ------ | ----------------------- |

| 1     | 0.25rem | 4px    | Minimal spacing (icons) |
| 2     | 0.5rem  | 8px    | Tight spacing (text)    |
| 3     | 0.75rem | 12px   | Compact elements        |
| 4     | 1rem    | 16px   | Standard spacing        |
| 5     | 1.25rem | 20px   | Medium spacing          |
| 6     | 1.5rem  | 24px   | Element separation      |
| 8     | 2rem    | 32px   | Section spacing         |
| 10    | 2.5rem  | 40px   | Large sections          |
| 12    | 3rem    | 48px   | Major section dividers  |
| 16    | 4rem    | 64px   | Page sections           |
| 20    | 5rem    | 80px   | Major page sections     |

#### Spacing Principles

- **Consistent**: Maintain consistent spacing for similar elements
- **Hierarchical**: Use larger spacing to separate major sections
- **Proportional**: Scale spacing based on screen size
- **Intentional**: Use spacing to create visual hierarchy and relationships

### Responsive Breakpoints

Our responsive system is designed to adapt gracefully across a wide range of devices.

#### Breakpoint Scale

| Name | Width     | Typical Device                           |
| ---- | --------- | ---------------------------------------- |
| xs   | < 640px   | Mobile phones (portrait)                 |
| sm   | >= 640px  | Mobile phones (landscape), small tablets |
| md   | >= 768px  | Tablets (portrait)                       |
| lg   | >= 1024px | Tablets (landscape), small laptops       |
| xl   | >= 1280px | Laptops, desktops                        |
| 2xl  | >= 1536px | Large desktops, monitors                 |

#### Responsive Guidelines

- **Mobile-First**: Design for mobile devices first, then enhance for larger screens
- **Fluid Layout**: Use flexible grids and containers that adapt to screen size
- **Consistent Experience**: Maintain consistent functionality across devices
- **Optimize Interactions**: Adapt interaction patterns for touch vs. mouse input

## Design Principles

Our design system is guided by five core principles:

### 1. Authentic & Friendly

Our designs feel personal and conversational, not corporate or generic. We use plain language, approachable visuals, and a tone that's helpful and human.

### 2. Speed & Simplicity

We prioritize intuitive flows and quick interactions. Our interfaces load quickly, respond immediately to user actions, and make complex travel planning feel straightforward.

### 3. Collaborative by Default

All core experiences are designed with multiple users in mind. We build features that make it easy for people to plan together, share ideas, and coordinate travel activities.

### 4. Balanced & Genuine

We reflect both the excitement of travel and the practical aspects of planning. Our design balances inspiration with useful tools that help people make real travel plans.

### 5. Responsive & Inclusive

We ensure a consistent experience across all devices and accessibility for all users. Our interfaces adapt to different screen sizes, devices, and browsing contexts.

## Accessibility Standards

Our design system follows WCAG 2.1 AA standards to ensure our platform is accessible to all users.

### Key Requirements

- **Color Contrast**: We maintain a minimum contrast ratio of 4.5:1 for normal text and 3:1 for large text and UI components
- **Keyboard Navigation**: All interactive elements are accessible via keyboard
- **Screen Readers**: We use semantic HTML and ARIA attributes to ensure screen reader compatibility
- **Focus Indicators**: All interactive elements have visible focus states
- **Text Resizing**: Our interfaces remain usable when text is resized up to 200%
- **Alternative Text**: All images and icons that convey meaning include appropriate alt text
- **Motion & Animation**: Users can reduce or disable non-essential motion
